## Overview

This document guides through the process of creating a simple HTTP GET request in the Ballerina Integrator using Devant's low-code platform. The request will return a simple JSON message: `"hello from Ballerina integrator"`.

**Key Objectives:**

- Add HTTP GET entry point
- Configure request/response handling
- Test the endpoint
- Verify response output

### Stage 3: Low-Code Interface Configuration

#### Step 3.1: Access Entry Points Panel
1. In the left sidebar, locate **"Entry Points"** section
2. Click on **"Entry Points"** to expand it
3. A toolbar appears with **"Add Entry Point"** button

**Purpose:** Entry points define the HTTP endpoints that will be exposed by the integration.

#### Step 3.2: Add HTTP GET Entry Point
1. Click **"Add Entry Point"** button
2. A configuration dialog appears (normally would load from low-code designer)
3. Select entry point type as **"HTTP Resource"**
4. Configure the following:
   - **HTTP Method:** GET
   - **Resource Path:** `/hello`
   - **Entry Point Name:** `HelloWorldGET`

**Screenshot:**  
![Add Entry Point Dialog](step_03_add_entry_point_dialog.png)

#### Step 3.3: Configure Response Mapping
- In the response mapping section:
  - **Content-Type:** application/json
  - **Response Body:** Map to JSON response with message

**Screenshot:**  
![Integrator BI Interface](step_03b_integrator_bi_interface.png)

---

### Stage 4: HTTP Listener Configuration

#### Step 4.1: Configure HTTP Listener
1. Click on **"Listeners"** section in the left panel
2. Add HTTP Listener with:
   - **Port:** 8080 (default)
   - **Protocol:** HTTP
   - **Binding:** localhost:8080

**Note:** The listener needs to be configured before entry points can be accessed via HTTP.

#### Step 4.2: Verify Connection Configuration
- Check **"Connections"** section for HTTP connection settings
- Ensure proper binding between listener and resources

---

### Stage 5: Response Configuration

#### Step 5.1: Configure Message Response
1. In the response mapping:
   - Create JSON response object
   - Add message field: `"message": "hello from Ballerina integrator"`

#### Step 5.2: Set Response Headers
- **Content-Type:** application/json
- **Status Code:** 200 (OK)
- **Cache-Control:** no-cache

---

### Stage 6: Deployment and Testing

#### Step 6.1: Deploy Integration
- Click **"Run Integration"** button in the toolbar
- Integration compiles and deploys locally
- Status bar shows deployment progress
- Confirms successful deployment

#### Step 6.2: Test the Endpoint
- Access the endpoint: `http://localhost:8080/hello`
- Expected Response:
  ```json
  {
    "message": "hello from Ballerina integrator"
  }
  ```

#### Step 6.3: Verify Response
- Status Code: **200 OK**
- Response Body: JSON object with message
- Response Headers: Content-Type: application/json

## Configuration Details

### HTTP GET Endpoint Configuration

```
Entry Point Configuration:
├── Name: HelloWorldGET
├── Type: HTTP Resource
├── Method: GET
├── Path: /hello
├── Request Parameters: None
├── Request Body: None (GET requests have no body)
└── Response Configuration:
    ├── Status Code: 200
    ├── Content-Type: application/json
    └── Message: "hello from Ballerina integrator"
```

### HTTP Listener Configuration

```
HTTP Listener Configuration:
├── Name: HttpListener
├── Type: HTTP Protocol
├── Host: localhost
├── Port: 8080
├── Connection Timeout: 60000 (ms)
├── Request Size Limit: 10485760 (bytes)
└── Binding:
    └── Resource: HelloWorldGET
        └── Path: /hello
            └── Methods: [GET]
```

## Troubleshooting

### Issue: Entry Point Not Responding

**Solution:**
1. Verify HTTP Listener is running on port 8080
2. Check firewall settings for localhost:8080
3. Ensure integration is deployed (check "Run Integration" status)

### Issue: 404 Not Found

**Solution:**
1. Verify resource path is exactly `/hello`
2. Check HTTP method is GET
3. Confirm entry point is linked to HTTP listener

### Issue: Incorrect Response Format

**Solution:**
1. Verify JSON response mapping
2. Check Content-Type header is application/json
3. Ensure message field name is exactly `"message"`
